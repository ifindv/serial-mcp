# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.0   | ✅        |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### How to Report

**Do not** open a public issue for security vulnerabilities.

Instead, send your report to:

- Email: ifindv@gmail.com

### What to Include

Please include the following information in your report:

1. Description of the vulnerability
2. Steps to reproduce the issue
3. Potential impact of the vulnerability
4. Suggested fix (if known)

### Response Time

We will:

- Acknowledge your report within 48 hours
- Provide a detailed response within 7 days
- Work with you to understand and fix the issue
- Release a patch as soon as possible

### Disclosure Policy

We will:

- Coordinate disclosure with you
- Announce the security fix publicly after it's released
- Credit you in the release notes (unless you prefer anonymity)

## Security Best Practices

When using this library:

- Only grant necessary permissions to serial ports
- Be cautious when handling untrusted data
- Validate all input before processing
- Use secure connections when transmitting sensitive data
- Keep the library updated to the latest version

## Dependencies

We regularly update dependencies to address security issues. If you find a vulnerable dependency:

1. Check if a fix is already available in the latest version
2. If not, open an issue with details about the vulnerability
